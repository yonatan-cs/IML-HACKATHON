בקצרה — מה בנינו ומה יצא

המשימה: CNN מאפס שמסווג תמונה ל־20 מחלקות, ועמיד למניפולציות ויזואליות (רקע/תאורה/צבע) שלא נראה מראש. קלט [B,3,224,224] מנורמל ImageNet, פלט מדד מחלקה [B] ב־0..19.

הפתרון: ResNet‑18 בצורתו + Squeeze‑Excitation (~11.28M פרמטרים), BatchNorm, אומן ב־AdamW + cosine + label smoothing. עמידות דרך אוגמנטציה כבדה ושומרת־תווית.

התוצאה: דיוק נקי 0.847, SCORE 0.906. ה־OOD יצא ~0.96 — גבוה מהנקי, כלומר המודל באמת אינווריאנטי ולא נשען על קיצורי דרך.

המשימה והניקוד

מודלים מודרניים מגיעים לדיוק גבוה על סט נקי — אבל לפעמים הם מסתמכים על רמזים מטעים. צפרדע על רקע ירוק תסווג נכון; אותה צפרדע על רקע כחול־שמיים פתאום מסווגת כספינה. המודל למד את הרקע, לא את האובייקט. זה קיצור דרך, והוא שביר. כל האתגר — להכריח את הרשת לזהות את האובייקט עצמו.

אנחנו צוות המחקר של חברת סטארט־אפ דמיונית, PixelPerfect: מודלים מדויקים וגם עמידים. מאמנים מאפס (בלי משקלים מאומנים מראש), 20 מחלקות, ועובדים טוב גם על נקי וגם על מעוות.

איך מנקדים — וזה כל המשחק

חצי הסטמה זה
50% In‑Domainתמונות נקיות, בהתפלגות כמו האימון
50% Out‑of‑Domainאוגמנטציות חבויות שלא קיבלנו (held‑out)

הציון משלב את שני החצאים שווה־בשווה. אי אפשר לקנות ציון רק עם דיוק נקי — חצי נשען על מה שלא ראינו. לכן אימדדנו את עצמנו על SCORE = 0.5·clean + 0.5·OOD, בדיוק כמו הgrader, ולא על דיוק נקי לבד.

חוקי הברזל של הgrader

נבדק אוטומטית. הפרה = פסילה או אפס. הכל נכתב כדי לעמוד בזה.

  • מאפס. אין משקלים מאומנים מראש, אין datasets חיצוניים, אין backbones, אין רשת בזמן אימון/טעינה/הערכה.
  • model.py מגדיר מחלקה בשם בדיוק ModelArchitecture (nn.Module), forward מחזיר לוגיטים [B,20], והקובץ עצמאי — מייבא רק torch/stdlib. הgrader משחזר ממנו את הרשת לבדו.
  • train.py שומר ל־weights.joblib כ־CPU state dict דרך joblib: joblib.dump(model.cpu().state_dict(), "weights.joblib"). (ה־docstring מזכיר torch.save — שגוי, joblib הוא הקובע.)

  • תוויות: labels.py מקור האמת. מדד 0..19 = מחלקות HF ממוינות. שמות תיקיות חייבים להתאים בדיוק.
  • הגשה: challenge2_<ids>/ = רק train.py, model.py, predict.py, weights.joblib, README. בלי dataset/venv/נתיבים מוחלטים.

הארכיטקטורה — ResNet‑18 + Squeeze‑Excitation

רשת בצורת ResNet‑18 (~11.28M פרמטרים), מאומנת מ־random init. הזרימה המרחבית: 224 → 112 → 56 → 56 → 28 → 14 → 7 → GAP → 20. כל החלקים נבחרו כדי להיות מוסברים בראיון.

המרכיבים, ולמה כל אחד

מרכיבמה הוא עושהלמה
StemConv 7×7 stride‑2 → BN → ReLU → MaxPool 3×3 stride‑2מקטין מהר 224→56, שדה קליטה גדול מההתחלה
Residual skipsy = f(x) + x (He 2016)מסלול ישיר לגרדיאנט אחורה → אפשר לאמן עומק מאפס בלי vanishing
BatchNormנרמול כל ערוץ על הbatch אחרי כל convמייצב את משטח ה־loss → רשת עמוקה מתאמנת נקי ומהר
Squeeze‑Excitationשער 0..1 לכל ערוץ (channel attention)מגביר ערוצים אינפורמטיביים, מדכא מסיחים — ישיר לעמידות
Dropout 0.4 + GAPhead דק: Pool→Dropout→Linear(512,20)רגולריזציה — 11M פרמטרים על ~16k תמונות = סכנת overfit

model.pynn.Module
class SEBlock(nn.Module):           # channel attention, reduction=16
    def forward(self, x):
        s = self.fc(self.pool(x))    # GAP -> 1x1(C->C/16) -> ReLU -> 1x1(->C) -> Sigmoid
        return x * s                  # שער לכל ערוץ [B,C,1,1], משדר על H,W

class ResidualBlock(nn.Module):
    def forward(self, x):
        identity = x if self.downsample is None else self.downsample(x)
        out = self.relu(self.bn1(self.conv1(x)))
        out = self.bn2(self.conv2(out))
        out = self.se(out)               # SE על F(x) לפני החיבור
        out = out + identity            # residual add
        return self.relu(out)

class ModelArchitecture(nn.Module):   # stem + 4 stages(2 blocks) + head
    def forward(self, x):
        x = self.stem(x)                          # 224->56
        x = self.stage1(x); x = self.stage2(x)      # 64 -> 128 (s2)
        x = self.stage3(x); x = self.stage4(x)      # 256 -> 512 (s2)
        x = self.pool(x)                          # AdaptiveAvgPool2d(1)
        return self.classifier(x)               # Dropout(0.4) -> Linear(512,20) = [B,20]
  • 8 בלוקים4 stages × 2 ResidualBlocks. רוחבים 64→128→256→512, הבלוק הראשון בכל stage (2–4) עושה stride‑2 ומכפיל ערוצים; ה־identity מותאם ב־1×1 conv+BN.
  • SE בכל בלוק8 בלוקי SE (~0.09M פרמטרים בלבד). s = sigmoid(W2·ReLU(W1·GAP(x))), out = x·s. שינויי רקע/צבע מפריעים בעיקר לחלק מהערוצים — SE לומד לדכא אותם.
  • BN אחרי כל conv20 שכבות BN. ה־conv עם bias=False כי ה־BN כבר נושא shift. ב־eval() משתמש ב־running stats קפואים → תחזית בלתי־תלויה בגודל batch.
  • initKaiming fan‑out (convs), BN weight=1/bias=0, Linear נורמלי קטן. He init מתאים ל־ReLU ומאפשר אימון מאפס.
  • GAPAdaptiveAvgPool2d(1) → הרשת מקבלת כל גודל קלט. אימנו ב־128 לזירוז, הgrader מזין 224, שניהם עובדים. אין FC גדול לפני ה־Linear → פחות פרמטרים ל־overfit.

העמידות — הלב של האתגר

חצי הציון הוא OOD. הכרחנו אינווריאנטיות דרך אוגמנטציה, ועשינו זאת אופליין: בנינו תאומים מאוגמנטים של תמונות האימון ושמרנו לדיסק.

למה אופליין (תאומים על הדיסק)

  • seeded per‑file → כל חברי הצוות מתאמנים על data זהה byte‑for‑byte.
  • ניתן לצפייה — אפשר לפתוח את התמונה המאוגמנטת ולראות מה קרה (טוב ל־error analysis).
  • leakage‑safe — שם זהה למקור → התאום יורש את ה־partition של המקור. תאום של תמונת‑test משמש רק כשהמקור ב־train.

המתודולוגיה — נבחנת בראיון

עקבנו אחרי "ML in Practice" מהתרגול, לא רק "תאמן רשת". זה חצי מהציון בראיון.

פיצול stratified, seeded, ו־curriculum מבוקר

5 חלקים per‑class (P0=40%, P1–P4=15%), seed=42, כל partition מייצג את כל ההתפלגות (אין דליפת מחלקות). Gated curriculum: שלב k מתאמן על P0..P(k-1), בודק את Pk פעם אחת בעודו unseen, ועוצר — המעבר לשלב הבא הוא החלטה ידנית. כל שלב מתחיל מ־random init טרי, לא warm‑start: בין שלבים שינינו ארכיטקטורה, ו־warm‑start בלתי אפשרי כשהרשת משתנה. הdata מצטבר, המודל מתאתחל. ה־Test המקומי נוגעים בו פעם אחת בסוף.

Baseline נאיבי כרצפה

רגרסיה לוגיסטית על פיקסלים מוקטנים נתנה ~0.22 — הרצפה שה־CNN חייב לנצח. אם לא מנצח, יש באג, לא בעיית כיוונון. ResNet עבר אותה בענק.

Error analysis ידני על Dev

משכנו דוגמאות שגויות, הסתכלנו בעיניים, חיפשנו דפוסים. מצאנו: acoustic_guitar מתבלבל עם laptop/mobile_phone/french_horn — כולם אובייקטים מלבניים כהים. המחלקות החזקות: goldfish, bald_eagle (~91%). אף פעם לא על ה־Test.

◆
בלי EDA נפרד: ויתרנו על שלב EDA פורמלי. שפיות מכנית (איזון מחלקות, שמות תיקיות) נאכפת אוטומטית בשלב הפיצול; ההסתכלות הויזואלית קורית ב־error analysis, על טעויות אמיתיות.

לפני הראיון — מה לדעת

היה פער קטן בין קבצי ההגשה, תיקנו אותו. כך זה עומד עכשיו:

◆
הכל עקבי: weights ↔ README ↔ train.py. אומת שה־weights.joblib שהוגש זהה בייט‑לבייט למודל שאומן עם התאומים האופליין (אותו checksum). הגרסה הראשונה של train.py שנארזה עשתה אוגמנטציה online ולא תאמה — החלפנו אותה ב־train.py עצמאי שבאמת בונה את התאומים המשוקללים (4/4/2/2/2/1, mean‑fill), בדיוק השיטה שייצרה את המשקלים. README נוקה גם (ירדו engine.py ו־SGD‑selectable). שלושתם מספרים אותו סיפור.
◆
rerun ≠ משקלים זהים. ליבות MPS/CUDA לא בִּיט‑דטרמיניסטיות, אז הרצה חוזרת של train.py לא תשחזר byte‑identical — לכן שומרים את weights.joblib בפועל. תשובה לגיטימית אם נשאלים "תריצו ותראו".
◆
נקודה חלשה ידועה: acoustic_guitar ↔ laptop/mobile_phone (אובייקטים מלבניים כהים). אם נשאלים "מה הייתם משפרים" — אוגמנטציה ספציפית לצורה/הקשר למחלקות האלה.

הגנה על כל רכיב

אין מצגת — בודקים הבנה. לכל רכיב, משפט שמסביר למה:

אם ישאלו על...התשובה
Residual skipsy=f(x)+x נותן לגרדיאנט מסלול ישיר אחורה → אפשר לאמן עומק מאפס בלי vanishing (He 2016).
BatchNormנרמול כל ערוץ על הbatch מייצב את משטח ה־loss; כמו LayerNorm ב־Transformers. ב־eval משתמש ב־running stats קפואים.
Squeeze‑Excitationchannel attention זול (~0.09M): שער 0..1 לכל ערוץ. מדכא ערוצים שמושפעים מרקע/צבע → ישיר לעמידות.
AdamWצעד אדפטיבי לכל פרמטר + weight decay מנותק → התכנסות מהירה ויציבה מאפס. SGD היה אופציה למעבר אחרון.
Label smoothingמטרות 0.9/0.1 מרסנות ביטחון יתר ומרתיעות התחייבות לקיצורי רקע/צבע.
למה לא background swapדורש סגמנטציה + אסטים חיצוניים → מפר מאפס/בלי data חיצוני. mean‑fill rotation = proxy חוקי.
מתודולוגיהsplit stratified seeded, פרוגרסיבי, Test נוגעים פעם אחת, baseline נאיבי 0.22 כרצפה, error analysis על Dev.
למה OOD > cleanאימון על תאומים מאוגמנטים הכריח אינווריאנטיות; המודל למד את האובייקט, אז מניפולציות לא מזיזות אותו.
◆
שורה תחתונה: כל בחירה ניתנת להסבר. עברת על המסמך — אתה מכיר את הארכיטקטורה, האימון, העמידות, התוצאות, והפערים. מוכן.
